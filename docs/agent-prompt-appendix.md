# Agent Prompt Appendix

The full LLM system prompt generated by `library.prompt()`. Paste into the backend agent (ADK / Claude API system prompt block) to enable structured OpenUI Lang responses.

---

## How to regenerate

```ts
// In any file that imports the library:
import { SYSTEM_PROMPT } from "@/openui/library"
console.warn(SYSTEM_PROMPT)
```

Or run the test which validates the prompt structure:
```bash
npm test -- openui-library
```

---

## Token budget guidance

| Section | Approx tokens |
|---|---|
| Preamble (MIA instructions) | ~120 |
| Component signatures + descriptions | ~800–1,200 (scales with 15 components) |
| Component group notes | ~200 |
| 3 few-shot examples | ~600 |
| **Total** | **~1,700–2,100 tokens** |

For context-window-constrained deployments, use `library.prompt({ examples: [] })` to drop few-shots (~600 tokens saved). The component signatures alone are sufficient for capable LLMs (Claude 3.5+).

**Trigger:** Budget Pace Guardrail fires on Paid Social WTD spend ($184K vs $240K plan, 77% pacing)

**Agent (OpenUI Lang — positional, what `<Renderer>` actually parses):**
```
root = LsStack("vertical", "md", [severity, alert, kpis, badge, action, steps, cta, chips])
  severity = LsSeverityBadge("high", "Budget Pace — High Severity")
  alert = LsInfoPanel("warning", "Paid Social is pacing 23% below plan for week 2 of Q4. At current rate, $340K will go unspent, forgoing an estimated $714K incremental revenue.", "Budget Pace Alert — Paid Social")
  kpis = LsKpiRow([{label: "Actual Spend (WTD)", value: "$184K", delta: -0.23, positive_direction: false, spark: [240, 232, 218, 205, 196, 184]}, {label: "Plan (WTD)", value: "$240K"}, {label: "Pace Gap", value: "$56K"}, {label: "Revenue at Risk", value: "$714K", delta: -0.23, positive_direction: false}])
```

## Invalid-Output Handling

The `<Renderer>` component handles invalid OpenUI Lang gracefully:

1. **Unknown component name** — logged to `onError` callback as `OpenUIError`, not rendered. The rest of the response continues rendering.
2. **Missing required prop** — component skipped, `onError` called with prop name and expected type.
3. **Type mismatch** — Zod coerces where possible (e.g. `"0.3"` → `0.3` for `delta`). Hard mismatches are skipped.
4. **Partial stream** — `isStreaming={true}` enables partial parse mode. Incomplete component calls render as much as available.

**Automated correction loop** (production):
```ts
<Renderer
  onError={(errors) => {
    if (errors.length > 0) {
      // Feed errors back to LLM for self-correction
      sendCorrectionMessage(errors.map(e => e.message).join("\n"))
    }
  }}
/>
```
